# Session note — 2026-05-01 21:04

**Live underpainting preview infrastructure.** Branch
`feature/underpainting-preview`, draft PR follows. Goal of this session:
shrink the painter feedback loop from 22 s (full pointillism) to under
200 ms (underpainting only) so future painter PRs and curation work can
iterate without the multi-second wait.

This is **infrastructure, not a painter feature.** No new artistic
capability. The pointillism pipeline is byte-identical to pre-PR;
verified by hash.

## What shipped

- **`src/snapshot.js` (new)** — `buildSnapshot()` extracted from
  `ControlsPanel.buildBindings`. Single source of truth for the
  StyleBindings shape across the existing "Test pointillism" trigger
  AND the new live preview, so they consume identical inputs (modulo
  render dimensions). All peeks are cache-only — paint-time / preview-
  time never block on Overpass / Open-Meteo round-trips.
- **`src/style/underpainting.js` (new)** — `renderUnderpainting(sourceCanvas, opts)`
  owns: working-canvas allocation, paintGround, paintCanopy,
  paintLandmarks, optional median-blur softening (`opts.softenEdges`,
  default true). Returns `{ canvas, srcData, timing }`. Each painter
  forks its own Mulberry32 from `opts.seed` via the same XOR salts
  Pointillism uses (canopy: `seed^0xC4_C4_C4_C4`, landmarks:
  `seed^0x14_14_14_14`) so canopy/landmark consumption doesn't shift
  the stroke-pass `rand`. **Kernel auto-scaling** added: `medianKernel:
  'auto'` (the new default) scales `11 × shortEdge / 3508` (A3 ref),
  floor 3, always odd. Pointillism passes an explicit kernel (11) so
  byte-parity at A3 holds.
- **`src/style/Pointillism.js` refactor** — `applyPointillism` now
  delegates the underpainting steps to `renderUnderpainting` and runs
  only palette / gradient / strokes itself. Pure refactor: same Snapshot
  in → same painting out. Verified by `scripts/parity-probe.js`
  (hash `cf15cf7b…80b39f`, identical before and after).
- **`src/ui/UnderpaintingPreviewPanel.js` (new)** — floating overlay
  bottom-right, 480×340 (long-edge cap, short scales to maintain export
  aspect ratio). Subscribes to `viewpoint:changed` (debounced 100 ms),
  `painter:changed`, `terrain:changed`, `time:changed`,
  `location:changed`, `sun:changed`, `export:changed`,
  `weather:fetched`, `weatherOverrides:changed`, `scene:ready`.
  Token-counter cancellation: every render bumps `renderToken`; older
  in-flight renders discard their result silently. Soften-edges toggle
  (default on). Render-time stat: `"<ms> · <dabs> dabs · <marks> mks"`.
  Close button calls `opts.onClose` (no state mutation).
- **`src/ui/ControlsPanel.js` (light edit)** — mounts the preview by
  default. New `<label class="pano-preview-toggle">` "Live preview"
  checkbox at the bottom of the sidebar. Local in-session toggle, no
  state-schema bump.
- **`src/ui/styles.css`** — overlay panel styling matches the existing
  `.pano-section` aesthetic.
- **No new state fields, no DATA-CONTRACTS.md schema bump.** The preview
  is a derived view of existing state. Show/hide is in-session UI
  ephemera; soften-edges toggle is in-panel local. If a future curation
  pass surfaces a need for persistence, the bump goes through the V2
  Step 5 pattern.

## Three probes (all pass)

### 1. Refactor parity (`scripts/parity-probe.js`)

Captures SHA-256 of full `applyPointillism` output on a fixed scene
(1600×1133, v1.4 expressionist preset, fixed seed `0xDEADBEEF`).
Re-run after each phase of the refactor — ALL three runs identical:

| Phase                   | sha256 (truncated)    | total ms |
| ----------------------- | --------------------- | -------- |
| Pre-refactor baseline   | `cf15cf7b…80b39f`     | 1426     |
| After snapshot.js       | `cf15cf7b…80b39f`     | 1340     |
| After underpainting.js  | `cf15cf7b…80b39f`     | 1349     |
| After kernel auto-scale | `cf15cf7b…80b39f`     | 1349     |

Byte-identical hash. Refactor preserved determinism.

### 2. Preview speed (`scripts/preview-speed-probe.js`)

A3-landscape preview-cap canvas (480 × 340), three scenes, 6 samples
each + warmup. Pass bar: median < 200 ms; star bar: < 100 ms.

| Scene  | Median  | Min     | Max     | Verdict |
| ------ | ------- | ------- | ------- | ------- |
| forest | 10.1 ms | 8.6 ms  | 10.4 ms | ★ PASS  |
| city   | 10.5 ms | 10.2 ms | 16.5 ms | ★ PASS  |
| combo  | 11.9 ms | 9.6 ms  | 14.3 ms | ★ PASS  |

10–12 ms median — **20× under the 200 ms pass bar, 8× under the 100 ms
star bar.** The auto-kernel scaling cut median-blur cost from ~21 ms
(11×11 at preview size) to ~9 ms (3×3 at preview size) and was the
single biggest win.

### 3. Preview-vs-full agreement (`scripts/preview-agreement-probe.js`)

Renders the same Snapshot at 480×340 AND at 4961×3508; downsamples the
A3 to preview size with bilinear; computes mean absolute error per
channel against the preview render.

Result: **MAE 0.18 / channel** (pass bar: < 14, ~5.5 % of 255).
First run (before kernel auto-scale) was 0.40 — still passing the
numerical bar but the visual divergence was real: the 11×11 kernel
over-blurred small landmarks at preview size, leaving silhouettes
invisible. After auto-scale (3×3 at preview), the horizon line and
landmark silhouettes are clearly visible in the preview, matching the
downsampled A3.

Outputs at `.iterations/2026-05-01-preview-agreement/` (preview.png,
full-down.png, full-thumb.png).

## Decisions made

- **UI placement: floating overlay, bottom-right, 480×340 default.**
  User chose this over right-rail / bottom-strip / sidebar in
  `AskUserQuestion` answer.
- **No new state fields.** Show/hide and soften-edges are in-panel /
  in-session. If curation needs persistence later, that's a deliberate
  bump.
- **`computeEffectiveDpi` is duplicated** in `underpainting.js` (small
  inline copy) instead of being imported from `Pointillism.js`. The
  alternative was a third shared module; one short formula's worth of
  duplication beats a new module just to dodge a circular import.
- **Kernel auto-scale lives in `renderUnderpainting`, not in the
  preview panel.** Means `medianKernel: 'auto'` is a property of the
  renderer that any caller benefits from. Pointillism explicitly pins
  11 to preserve byte-parity; future callers can stay on 'auto'.
- **`renderUnderpainting` allocates a working canvas even when no
  bindings are present** (small extra `drawImage` cost). Keeps the
  return canvas independent of the input — the preview can hold onto
  it without the 3D viewer's next frame overwriting it. Byte-parity
  preserved because the data ends up identical.

## Out of scope (deferred per the brief)

- **OSM-fetched event** to refresh the preview when an Overpass cache
  warm lands. The brief restricts edits to `src/style/`, `src/ui/`, and
  the new `src/snapshot.js`; emitting a new event from
  `src/scene/SceneManager.js` or `src/osm/index.js` would touch outside
  modules. Today the preview re-renders on `scene:ready` and on every
  user-driven event; OSM cache landing is invisible until the user
  nudges anything. Acceptable for v0.
- **Sky-gradient and DEM-skyline rendering inside `renderUnderpainting`.**
  STRATEGY-V2 lists these as Stage 1 underpainting steps but they
  don't exist yet (`SkylineCaster.getSkyline()` is documented as
  unbuilt). Today the source canvas comes from the WebGL viewer
  snapshot — same as pre-PR Pointillism. Future PR can replace that
  with synthesised sky+skyline once `SkylineCaster` lands.
- **Test pointillism / Save image visual QA on the refactor.** Hash
  parity proves byte-identical output but hasn't been re-verified in
  the browser. The user's `npm run dev` flow remains the canonical
  visual gate.
- **Persistence of show/hide and soften-edges toggle.** Per the brief,
  not adding state fields unless a need is surfaced.
- **PR #12 canopy-density curation.** Stays parked for a separate PR.
